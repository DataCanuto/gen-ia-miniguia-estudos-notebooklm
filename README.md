# Miniguia de Estudos com NotebookLM

## UML e Diagramas de Caso de Uso

Projeto desenvolvido como parte do desafio da **DIO (Digital Innovation One)** com o objetivo de explorar o uso de Inteligência Artificial como ferramenta de aprendizagem ativa.

Neste projeto foi utilizado o **NotebookLM** para organizar fontes de estudo, realizar análises com IA e construir um caderno temático estruturado sobre **UML e Diagramas de Caso de Uso**, um dos fundamentos da modelagem de sistemas em engenharia de software.

---

# 1. Contexto e Objetivos

A **UML (Unified Modeling Language)** é uma linguagem padrão amplamente utilizada para modelar sistemas de software. Entre seus diversos diagramas, os **diagramas de caso de uso** são utilizados para representar interações entre usuários e sistemas, ajudando a compreender requisitos funcionais.

Este projeto tem como objetivos:

* compreender os fundamentos da UML
* estudar diagramas de caso de uso e sua aplicação
* utilizar IA como ferramenta de apoio à aprendizagem
* aplicar engenharia de prompts para extrair conhecimento das fontes
* organizar o conteúdo em um miniguia de estudo reutilizável

# 3. Engenharia de Prompts

Durante o uso do NotebookLM foram testados diferentes prompts para explorar melhor o conteúdo das fontes.

## Prompt 1

Explique o que é UML e qual seu papel no desenvolvimento de software.

Resultado:

A UML é uma linguagem de modelagem padronizada utilizada para visualizar, especificar, construir e documentar sistemas de software. Ela permite representar diferentes aspectos de um sistema por meio de diagramas.

---

## Prompt 2

Explique o que é um diagrama de caso de uso.

Resultado:

Um diagrama de caso de uso representa as interações entre usuários (atores) e um sistema. Ele ajuda a identificar funcionalidades do sistema e como os usuários irão interagir com ele.

---

## Prompt 3

Quais são os principais elementos de um diagrama de caso de uso?

Resultado:

Os principais elementos são:

* atores
* casos de uso
* sistema
* relações entre casos de uso (include, extend)

---

## Prompt 4

Quais são as vantagens de utilizar UML no desenvolvimento de software?

Resultado:

Entre as principais vantagens estão:

* melhor comunicação entre equipes
* documentação estruturada do sistema
* visualização da arquitetura do software
* apoio ao levantamento de requisitos

## Troubleshooting (Dificuldades encontradas)

# 4. Miniguia de Estudo

## Resumo Estruturado

A UML é uma linguagem de modelagem visual usada para representar sistemas de software.

Seus principais objetivos são:

* facilitar a compreensão de sistemas complexos
* documentar requisitos e arquitetura
* melhorar a comunicação entre analistas, desenvolvedores e stakeholders

Entre os diagramas mais utilizados da UML estão:

* diagrama de caso de uso
* diagrama de classes
* diagrama de sequência
* diagrama de atividades

O **diagrama de caso de uso** é especialmente útil nas fases iniciais do desenvolvimento, pois permite identificar as funcionalidades que o sistema deve oferecer aos usuários.

---

## Glossário

UML (Unified Modeling Language)
Linguagem padronizada para modelagem de sistemas de software.

Caso de Uso
Descrição de uma interação entre um usuário e um sistema para atingir um objetivo específico.

Ator
Representa um usuário ou sistema externo que interage com o sistema modelado.

Diagrama de Caso de Uso
Representação visual das funcionalidades de um sistema e das interações com os atores.

Include
Relação que indica que um caso de uso sempre inclui outro.

Extend
Relação que indica que um caso de uso pode estender outro em determinadas condições.
